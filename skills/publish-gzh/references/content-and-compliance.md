# 内容与合规基线

## 单一事实源

- 公众号文章 frontmatter 作者固定为 `桥博士`，不能写 `宽论`。
- 微信草稿主题固定为 `orangeheart`。
- 宽论/QMACD 的定位是量化分析技术、技术方法论和工具使用教育。
- 不把它描述成投资顾问、荐股、账户诊断、买卖建议或保证收益服务。

这些规则覆盖旧 README、旧文章、旧 Agent 或旧 prompt 中的冲突说法。

## 事实使用规则

可以使用的内容：

- 用户提供且可核验的原始材料。
- 项目内带来源的批准事实清单。
- 官方公开材料中能定位到具体出处的事实。
- 明确标注为“技术演示，非投资建议”的虚拟技术示例。

不能使用的内容：

- 没有逐条来源的盈利数字、竞赛名次、媒体背书或从业年限。
- 虚构学员、朋友、读者故事或调查结果。
- 把社区规模推导成会员盈利。
- 把历史结果写成未来承诺。
- 真实身份证号、手机号、账号凭据或其他不必要个人信息。

建议在文章旁维护一个不发布的来源清单：

```yaml
claims:
  - claim: ...
    source: <path-or-url>
    location: <page-section-or-line>
    status: verified | unresolved
```

`unresolved` 事实不得进入待发布文章。

## 金融内容红线

出现以下含义时直接判定发布失败：

- “跟着我们就能赚钱”或同义承诺。
- 推荐具体股票、具体买卖点或个性化账户建议。
- 保本、稳赚、高胜率无风险等保证。
- 通过未注明条件的历史数字暗示未来结果。

本 Skill 面向宽论/QMACD 的交易与量化内容，因此每篇待发布文章都必须包含：

```markdown
*免责声明：本文介绍的是量化分析技术与方法论，不构成投资建议。投资有风险，入市需谨慎。*
```

## 中文写作标准

- 结论先行，先回答读者为什么要看。
- 用短段落；一个段落只承担一个功能。
- 第一次出现专业词时先解释，再使用缩写或术语。
- 少用“首先、其次、最后、综上所述”等空过渡。
- 用具体场景和可核验证据代替抽象自夸。
- 标题不超过 64 个字符，并在 frontmatter 中保存最终标题。
- 不输出“下面是文章”等对正文无用的说明。

## 参考稿与版权边界

可以学习结构功能，不能保存或分发无授权全文。重写时记录：

- 标题用了什么注意力机制。
- 第一段解决了什么心理问题。
- 论证节奏如何变化。
- 哪些情绪节点可迁移。
- 哪些句子、人物和数据必须舍弃。

最终文字必须独立表达，不能一比一模仿标题和第一段。
